# Web dashboard (React + Vite)

An interactive, **fully static** dashboard for the Cullinan Sand geotechnical
dataset — plate sinkage, Bevameter shear, and triaxial compression. It compiles
to plain HTML/CSS/JS with **no backend**: all data is loaded from static JSON
files under `public/data/`. Deployable to GitHub Pages as-is.

- **Stack:** React 18, Vite 5, [Recharts](https://recharts.org) for plots,
  Tailwind CSS v4 for styling.
- **Theme:** a dark "geotechnical instrument" design — deep graphite surfaces, a
  sand-ochre signature accent, monospace data readouts, and a CVD-safe
  categorical palette validated against the dark surface.

---

## Quick look (no build)

The compiled site is committed in [`dist/`](dist/). Because the app fetches JSON,
browsers block it over `file://` — serve it over HTTP:

- **Windows:** double-click [`serve_dashboard.bat`](serve_dashboard.bat) — it
  starts a plain HTTP server on `http://localhost:8000` and opens your browser.
- **Any OS:** `cd dist && python -m http.server 8000`, then open
  <http://localhost:8000>.

## Develop & build

> **Note on this repository's location.** This project lives on a Google Drive
> synced folder, where `npm install` is unreliable (Drive locks the many small
> files npm writes). Install/build from a **normal local disk** — e.g. after a
> `git clone` to a non-synced path. `node_modules/` is git-ignored and absent
> from the archive by design; the committed `dist/` is the runnable artifact.

```bash
npm install     # fetch React, Vite, Recharts, Tailwind
npm run dev      # dev server with hot reload
npm run build    # static production build → dist/
npm run preview  # serve the built dist/ locally
```

To regenerate the data the app consumes, run the Python builder (from `python/`):

```bash
python scripts/build_web_data.py
```

It writes `public/data/*.json` (also copied to `data/processed/web_json/`). Vite
copies `public/` into `dist/` at build time, so `dist/data/` ships with the app.

## Data it loads (`public/data/`)

Generated by `../python/scripts/build_web_data.py` — curves are decimated for
display (≤ 2000 points/test, shape-preserving LTTB); the full-resolution data
lives in `../data/raw/`.

| File | Contents |
|------|----------|
| `index.json` | dataset catalogue: `{ datasets:[{type,title,file,n_tests}], max_points }` |
| `plate_sinkage.json` | 13 tests · columns `sinkage`,`pressure` · grouped by `method` |
| `shear.json` | 4 tests · columns `Displacement`,`shear_stress` · grouped by `Method` |
| `triaxial.json` | 16 tests · columns `axial_strain`,`q`,`p_prime`,`e` · 3 linked panels |

Each `<type>.json` has `axes` (labels/units), and `tests[]` with `test_id`,
`scalars` (per-test metadata), and row-aligned `columns` (`null` for gaps).

## Features

- **Tabs** per test type (keyboard-navigable, WAI-ARIA).
- **Plate sinkage:** pressure–sinkage curves coloured by compaction method,
  per-method filter checkboxes, a toggleable 100 kPa nominal-pressure reference
  line (secondary force axis note: force [kN] = pressure [kPa] × 0.018), legend,
  metadata table, and client-side "download visible series as CSV".
- **Shear:** multi-select tests, coloured by method (consistent with the plate
  tab), rich legend `Method | σₙ | density`, table, CSV download.
- **Triaxial:** dropdown to pick a primary test, three linked panels
  (q–εₐ, q–p′, and e–p′ with a log-scale x-axis), optional overlay of other
  tests on the q–p′ panel, a 200-row data table, and CSV download.
- All charts plot **every available point** and show a hover crosshair + tooltip.

## Deploy to GitHub Pages

`vite.config.js` sets `base: './'` (relative), so the build works at any Pages
subpath and when opened locally. Two options:

1. **Publish the committed build** — serve `dist/` (e.g. via the workflow at
   [`.github/workflows/deploy-pages.yml`](../.github/workflows/deploy-pages.yml),
   which rebuilds and deploys on push).
2. **Build in CI** — `npm ci && npm run build`, then deploy `dist/` with
   `actions/deploy-pages`.

Enable it under repo **Settings → Pages → Source: GitHub Actions**.
